Documentation

Complete guide to integrating and using the Import SDK in your applications

Getting Started

Installation

Install the Import SDK using your preferred package manager:

npm install import-sdk
yarn add import-sdk
pnpm add import-sdk

Basic Setup

Create a container element and initialize the SDK:

<!-- HTML -->
<div id="import-container"></div>

<!-- JavaScript -->
<script src="import-sdk.js"></script>
<script>
  const sdk = ImportSDK.init(document.getElementById('import-container'), {
    apiEndpoint: 'http://localhost:3000/api/import',
    chunkSize: 100
  });
</script>

Configuration Options

Option Type Default Description
apiEndpoint string required API endpoint for sending import data
chunkSize number 100 Number of rows to process in each batch
concurrency number 1 Number of concurrent batch requests
waitBetweenChunks number 0 Delay in milliseconds between chunk batches
fieldMapping object {} Map CSV headers to field names
transformers object {} Functions to transform field values
filters object {} Functions to filter rows before processing
resultExport array [] Data types to collect for export

Advanced Configuration

Concurrency & Flow Control

Control the speed and load of the import process:

ImportSDK.init(container, {
    chunkSize: 100,
    concurrency: 5,        // Send 5 chunks (500 records) in parallel
    waitBetweenChunks: 1000 // Wait 1 second after each batch of 5 chunks
});

Concurrency: The SDK will take concurrency chunks from the buffer and send them simultaneously using Promise.all.

Wait Time: After a batch of parallel requests completes, the SDK will wait for waitBetweenChunks milliseconds before processing the next batch.

Custom Headers & Fetch Options

Customize HTTP headers and fetch options for the default send handler:

ImportSDK.init(container, {
    headers: {
        'Authorization': 'Bearer YOUR_TOKEN_HERE',
        'X-Custom-Header': 'custom-value'
    },
    fetchOptions: {
        credentials: 'include',
        mode: 'cors'
    }
});

Custom Send Handler

Use a custom function to send data to your API:

ImportSDK.init(container, {
    sendHandler: async (batch, config) => {
        const response = await fetch(config.apiEndpoint, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'Authorization': `Bearer ${myToken}`
            },
            body: JSON.stringify({ items: batch })
        });
        
        const data = await response.json();
        
        return {
            success: data.imported || 0,
            errors: data.errors || []
        };
    }
});

Internationalization (i18n)

Configure the SDK for multiple languages:

ImportSDK.init(container, {

    translations: {
        fr: {

CSV Normalization Layer

Automatically fix common issues in real-world CSV files before parsing. This feature is enabled by default.

Option Default Description
enabled true Enable/disable the entire normalization layer
trimBOM true Remove Byte Order Mark (BOM) from file start
normalizeLineEndings true Convert CRLF and CR to LF line endings
autoDetectDelimiter true Auto-detect comma, semicolon, tab, or pipe delimiters
removeUnicodeJunk true Remove invisible Unicode and control characters
stripEmptyLines true Remove empty lines (preserves header)
sanitizeHeaders true Clean header row of problematic characters 
// Enable all normalization (default)
ImportSDK.init(container, {
    csvNormalization: {
        enabled: true
    }
});

// Selective normalization
ImportSDK.init(container, {
    csvNormalization: {
        enabled: true,
        trimBOM: true,
        normalizeLineEndings: true,
        autoDetectDelimiter: true,
        removeUnicodeJunk: false,    // Keep Unicode as-is
        stripEmptyLines: true,
        sanitizeHeaders: false       // Keep headers as-is
    }
});

Data Filtering

Configure filters to exclude rows based on custom logic before validation and import:

ImportSDK.init(container, {
    filters: {
        // Skip rows where city is empty
        skipEmptyCity: (row) => row.city && row.city.trim() !== '',
        
        // Only process specific tank types
        allowedTypes: (row) => ['1', '2', '3'].includes(row.typeId),
        
        // Skip test data
        excludeTest: (row) => !row.tankNumber.startsWith('TEST_'),
        
        // Geographic filter
        parisOnly: (row) => row.city === 'Paris'
    }
});

Filter Function Signature:

filterName: (transformedRow, originalRow, rowIndex) => boolean

Parameters:

  • transformedRow: Row data after field mapping and transformers
  • originalRow: Original CSV row data before transformation
  • rowIndex: Zero-based row number in the CSV file

Multiple File Mappings

Configure different field mappings for different CSV formats. The SDK automatically selects the appropriate mapping based on the filename.

ImportSDK.init(container, {
    fileMappings: [
        {
            name: 'Standard Format',
            pattern: /test\.csv$/,
            fieldMapping: {
                'Tank ID': 'tankNumber',
                'Type': 'typeId'
            }
        },
        {
            name: 'Custom Format',
            pattern: /test2\.csv$/,
            fieldMapping: {
                'ContainerID': 'tankNumber',
                'BinType': 'typeId'
            },
            transformers: {
                typeId: (v) => parseInt(v)
            }
        },
        {
            name: 'Legacy Format',
            pattern: 'legacy',  // Matches any filename containing 'legacy'
            fieldMapping: {
                'ContainerID': 'tankNumber',
                'BinType': 'typeId'
            }
        }
    ],
    
    // Fallback mapping (used if no pattern matches)
    fieldMapping: {},
    transformers: {}
});

How it works:

  1. User selects a file
  2. SDK checks fileMappings array for matching pattern
  3. First matching mapping is used
  4. If no match, falls back to default fieldMapping and transformers
  5. Logs show which mapping was selected

Plugin System

Plugin Registration

Register plugins using the static use() method before initializing the SDK:

// Field-level plugin
ImportSDK.use({
    name: 'dateNormalizer',
    type: 'field',
    transform(value, fieldName, row, config) {
        if (fieldName.includes('Date') && typeof value === 'string') {
            return new Date(value).toISOString();
        }
        return value;
    }
});

// Row-level plugin
ImportSDK.use({
    name: 'coordinateValidator',
    type: 'row',
    validate(row, config) {
        const lat = parseFloat(row.latitude);
        const lng = parseFloat(row.longitude);
        
        if (lat < -90 || lat > 90 || lng < -180 || lng > 180) {
            return { isValid: false, error: 'Invalid GPS coordinates' };
        }
        return { isValid: true };
    }
});

Plugin Types

Type Level Use Cases
Field Individual field values Date parsing, number formatting, text normalization
Row Complete data rows Cross-field validation, row enrichment, conditional logic
Batch Groups of rows Batch processing, API optimization, aggregation

Plugin Management

// List all plugins
console.log(ImportSDK.getPlugins());

// Get plugins by type
const fieldPlugins = ImportSDK.getPluginsByType('field');
const batchPlugins = ImportSDK.getPluginsByType('batch');

// Remove a specific plugin
ImportSDK.removePlugin('dateNormalizer');

// Clear all plugins
ImportSDK.clearPlugins();

Import Execution Metrics

Monitor and optimize import performance with comprehensive execution metrics:

Timing Metrics

  • totalDuration - Complete import duration
  • parseDuration - CSV parsing time
  • normalizationTime - CSV normalization processing time
  • avgRowProcessingTime - Average time per row
  • avgChunkLatency - Average time per batch/chunk
  • avgApiLatency - Average API call response time

Throughput Metrics

  • rowsPerSecond - Processing rate (rows/second)
  • throughput - Data processing rate (bytes/second)
  • efficiency - Actual vs theoretical processing efficiency (%)
  • successRate - Percentage of successfully processed rows
  • errorRate - Percentage of rows with errors
  • peakConcurrency - Maximum simultaneous API calls
ImportSDK.init(container, {
    onMetrics: (metrics) => {
        console.log('Performance Metrics:', {
            duration: `${metrics.totalDuration}ms`,
            throughput: `${metrics.rowsPerSecond} rows/sec`,
            efficiency: `${metrics.efficiency}%`,
            apiLatency: `${metrics.avgApiLatency}ms`
        });
    }
});

API Reference

ImportSDK.init(container, config)

Initialize a new Import SDK instance.

Parameters:

  • container (HTMLElement) - DOM element to render the UI
  • config (Object) - Configuration options
const sdk = ImportSDK.init(document.getElementById('container'), {
  apiEndpoint: 'http://localhost:3000/api/import',
  chunkSize: 100,
  fieldMapping: {
    'Name': 'name',
    'Email': 'email'
  },
  transformers: {
    email: (value) => value.toLowerCase().trim()
  }
});

Event Handlers

Configure callbacks to monitor import progress:

const sdk = ImportSDK.init(container, {
  onProgress: (stats) => {
    console.log(`Progress: ${stats.totalCount} rows processed`);
  },
  onComplete: (results) => {
    console.log(`Import complete: ${results.successCount} successful, ${results.errorCount} errors`);
  },
  onError: (error) => {
    console.error('Import error:', error);
  },
  onMetrics: (metrics) => {
    console.log(`Performance: ${metrics.rowsPerSecond} rows/sec`);
  }
});

Export Functions

Export processed data in various formats:

Available Export Methods:

  • sdk.exportErrors() - Download error rows as CSV
  • sdk.exportSuccess() - Download successful rows as CSV
  • sdk.exportFiltered() - Download filtered rows as CSV
  • sdk.exportLogs() - Download logs as JSON
  • sdk.exportAll() - Download all data types
// Enable export options
const sdk = ImportSDK.init(container, {
  resultExport: ['errors', 'success', 'logs']
});

// Export will be available after import completes
sdk.exportErrors(); // Downloads errors.csv

Examples

Basic CSV Import

Simple setup with default configuration:

const sdk = ImportSDK.init(document.getElementById('import-container'), {
  apiEndpoint: '/api/import',
  chunkSize: 50
});

Advanced Configuration

Full configuration with field mapping and validation:

const sdk = ImportSDK.init(document.getElementById('import-container'), {
  apiEndpoint: '/api/users/import',
  chunkSize: 100,
  concurrency: 3,
  updateByTankNumber: false,
  fieldMapping: {
    'First Name': 'firstName',
    'Last Name': 'lastName',
    'Email Address': 'email',
    'Phone': 'phoneNumber'
  },
  transformers: {
    email: (value) => value.toLowerCase().trim(),
    phoneNumber: (value) => value.replace(/[^\d]/g, '')
  },
  filters: {
    email: (value) => value && value.includes('@')
  },
  resultExport: ['errors', 'success', 'logs'],
  csvNormalization: {
    enabled: true,
    autoDetectDelimiter: true,
    removeUnicodeJunk: true
  },
  onProgress: (stats) => {
    updateProgressBar(stats.totalCount);
  },
  onComplete: (results) => {
    showNotification('Import completed successfully!');
  }
});

File-Specific Mappings

Configure different mappings based on filename patterns:

const sdk = ImportSDK.init(document.getElementById('import-container'), {
  apiEndpoint: '/api/import',
  fileMappings: [
    {
      pattern: /users.*\.csv$/,
      name: 'Users Import',
      fieldMapping: {
        'Name': 'fullName',
        'Email': 'emailAddress'
      },
      transformers: {
        emailAddress: (value) => value.toLowerCase()
      }
    },
    {
      pattern: /products.*\.csv$/,
      name: 'Products Import',
      fieldMapping: {
        'Product Name': 'name',
        'SKU': 'productCode',
        'Price': 'unitPrice'
      },
      transformers: {
        unitPrice: (value) => parseFloat(value)
      }
    }
  ]
});
